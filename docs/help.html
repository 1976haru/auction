이 앱은 무엇

법원경매·온비드 공매 매물을 검색·필터·차트·자연어로 빠르게 탐색하는 정적 데모입니다. 실제 거래 대상이 아닌 mock 데이터로 동작하며, 외부 API 호출 없이 클라이언트(브라우저)에서 모든 처리를 합니다.

PWA로 폰 홈 화면에 설치하면 standalone 앱처럼 동작하고, 오프라인에서도 마지막 로드된 데이터가 표시됩니다.

빠른 시작 (6단계)

  1. 검색바에 주소·사건번호·키워드를 입력합니다 (예: 강남, 2025타경, 임차인). 입력하는 동안 자동으로 필터링됩니다.
  2. 빠른 메뉴 칩(전체/경매/공매/아파트/...) 또는 프리셋 칩(🚀 고수익 저위험, ⏰ 임박 후보 등)을 한 번 눌러 결과를 좁힙니다.
  3. 관심 가는 매물 카드를 탭해 상세 모달을 엽니다. 시세 트렌드 곡선, 입찰가 시뮬레이터, 점수 분해, 위험·체크리스트, 비슷한 매물을 한 화면에서 봅니다.
  4. ★ 관심·📝 메모로 매물별 정리, ⇆ 비교 트레이로 2~5건을 나란히 봅니다.
  5. 🔗 공유로 매물·결과를 카톡 등으로 보냅니다 (받은 사람도 같은 화면).
  6. ⚙ 설정에서 데이터 백업/복원, 🌙로 다크 모드, 🖨로 인쇄/PDF.

검색 · 탐색

통합 검색

검색바 한 칸에서 주소·사건번호·물건명·키워드를 모두 검색합니다. 200ms 디바운스로 입력하는 즉시 결과가 갱신됩니다. Enter나 검색 버튼을 누르면 최근 검색어에도 저장됩니다.

최근 검색어

검색바를 클릭하면 localStorage에 저장된 최근 8개 검색어가 드롭다운으로 노출됩니다. 행 클릭 시 그 검색어로 즉시 재검색되고, 행의 ×로 개별 삭제·하단의 "전체 지우기"로 일괄 삭제할 수 있습니다.

빠른 메뉴 칩

전체 / ★ 내 관심 / 📝 메모 / 👁 최근 본 / 🆕 변화 / 경매 / 공매 / 아파트 / 오피스텔 / 빌라 / 상가 / 토지 / 고수익 / 저위험 / 입찰임박 / 고위험. 한 번에 하나만 활성화됩니다.

필터 프리셋 (1클릭 다중 필터)

  • ⭐ 추천 후보: 점수 60+, 추천점수 정렬
  • 🚀 고수익 저위험: 위험 낮음 + 차익 큰순
  • ⏰ 임박 후보: D-7 이내 + 임박순
  • 🏠 서울 아파트: 지역 + 종류 + 추천 정렬
  • 🥇 A등급만: A 등급 + 점수순

프리셋을 누르면 토스트가 뜨고 "되돌리기"로 직전 상태로 복귀할 수 있습니다.

상세 필터

지역, 물건종류, 경매·공매, 최저가 범위, 유찰횟수, 입찰기일(D-N), 위험도, 추천등급으로 좁힙니다. 정렬은 추천점수·차익·수익률·기일·최저가·위험 6종.

AI 에이전트 검색

"시세차익 큰 물건 5개", "서울 아파트 중 위험 낮은 물건", "입찰기일 7일 이내 후보", "오늘 뭐부터 봐야 돼?" 같은 한국어 자연어를 rule-based로 해석해 즉시 재정렬·재필터합니다. 외부 API 호출은 하지 않습니다.

URL 동기화 / 결과 공유

검색·필터·정렬·뷰가 querystring에 반영됩니다. 새로고침해도 그대로 복원되며, 헤더의 🔗 버튼으로 현재 결과 URL을 공유 시트로 보낼 수 있습니다.

매물 상세 · 시뮬레이터

기본 정보 + 시세 트렌드

구분·사건번호·주소·종류·감정가·최저가·예상시세·유찰·입찰기일·위험·등급·신뢰도가 한눈에. 12개월 시세 sparkline에 감정가/최저가/추정시세 점선 기준선이 함께 표시됩니다.

예상 입찰가 + 시뮬레이터

정적 보수/기준/공격 카드 아래에 인터랙티브 시뮬레이터가 있습니다. 슬라이더를 좌우로 끌면:

  • 입찰가별 차익·ROI·총 비용이 즉시 계산됨
  • 입찰가-차익 곡선이 슬라이더 위에 표시 (양/음 영역 음영, 손익분기점 자동 표시)
  • 현재 입찰가 마커 색이 양수=초록 / 음수=빨강으로 즉시 전환

비용은 취득세(아파트·오피스텔 6억↓ 1%, 9억까지 3.5%, 9억↑ 3% / 그 외 3.5%) + 수리 500만 + 명도 300만 + 금융(연 4% × 6개월) 가정입니다.

위험 분석 · 추가 확인사항 · 다음 액션

매물별 위험 키워드(임차인·대항력·유치권 등)와 키워드별 체크리스트, 다음에 해야 할 액션이 정리됩니다.

추천 점수 분해

추천 점수가 어떻게 만들어졌는지 5개 가로 막대로 분해 — 차익(35) / 수익률(25) / 위험(20) / 신뢰도(10) / 기일 임박(10) + 합계.

최근 변화 (7일)

최저가·유찰·기일·상태 변화 이력 + 카드 헤더의 색깔 배지(🆕 신규 / ⬇ 인하 / 📅 기일 / 🔄 유찰 / ⚠ 상태).

비슷한 매물

같은 지역(30) + 종류(25) + 가격대 ±20%(15) + 등급(6) 점수로 매칭된 다른 매물 3건. 카드 탭 시 같은 모달 안에서 매끄럽게 다른 매물로 이동합니다.

개인화 (★ / 📝 / ⇆ / 👁)

모든 개인 데이터는 자기 폰 브라우저(localStorage)에만 저장됩니다. 서버로 전송되지 않습니다.

★ 관심 매물

카드/모달의 ★ 버튼으로 토글. "★ 내 관심" 칩으로 모아 보기. 기본 보기에선 ★ 매물이 어떤 정렬에서도 자동으로 맨 위에 표시됩니다.

📝 매물 메모

모달 하단 textarea에 메모를 적으면 600ms 디바운스 + blur 시 자동 저장됩니다. 메모가 있는 매물 카드 헤더에 📝 배지가 붙고, "📝 메모" 칩으로 모아 볼 수 있습니다.

⇆ 비교 트레이

카드의 ⇆ 토글이나 카드를 0.5초 길게 눌러 2~5건을 트레이에 담습니다. 트레이에서 "비교하기"를 누르면 표 형식의 비교 모달이 열리고, 등급·점수·차익·ROI·신뢰도 등에서 ▲ 최고 / ▼ 최저가 자동 마킹됩니다. 🖨로 비교표 PDF 저장도 가능합니다.

👁 최근 본 매물

매물 모달을 열 때마다 자동 기록되며 최대 20건이 저장됩니다. "👁 최근 본" 칩으로 본 순서대로 정렬해 빠르게 다시 찾을 수 있습니다.

✨ 맞춤 추천

★/📝/👁 시그널이 3건 이상 모이면 추천 TOP 5 아래에 "맞춤 추천" 카드가 자동 노출됩니다. 사용자 선호 지역·종류·가격대를 분석해 비슷한 매물 5개를 점수순으로 보여줍니다.

공유 · 내보내기

🔗 매물 공유

매물 모달의 🔗 버튼은 시스템 공유 시트(카톡/메시지/AirDrop 등)를 열고 제목·요약·URL을 함께 보냅니다. URL에는 #item-N 해시 딥링크가 포함돼 있어 받은 사람이 링크를 누르면 바로 그 매물 모달이 열립니다.

🔗 결과 공유

전체 물건 헤더의 🔗 버튼은 현재 적용된 검색·필터·정렬 URL을 한 줄 요약과 함께 공유합니다. 받은 사람도 같은 검색 화면을 그대로 볼 수 있습니다.

CSV / JSON 다운로드

현재 필터된 결과를 23개 컬럼 CSV(UTF-8 BOM, 엑셀에서 한글 깨지지 않음) 또는 메타 포함 JSON으로 저장합니다. 파일명에 timestamp가 포함됩니다.

🖨 매물·비교 PDF

매물 모달과 비교 모달의 🖨 버튼을 누르면 그 모달만 풀페이지로 인쇄됩니다. iOS Safari·Android Chrome의 시스템 인쇄 시트에서 "PDF로 저장"을 선택하면 파일이 저장됩니다.

📡 RSS 피드

/feed.xml — 추천 점수순 상위 50건의 RSS 피드. Feedly·Inoreader 같은 외부 피드 리더에 등록하면 매일 자동 갱신된 추천을 받을 수 있습니다.

시스템 (다크 · PWA · 단축키)

다크 모드

OS의 prefers-color-scheme: dark를 자동 감지하며, 검색바의 🌙/☀️ 버튼으로 명시 토글할 수 있습니다. 명시 선택은 OS 설정보다 우선하며 localStorage에 영구 저장됩니다.

PWA 설치

Android Chrome: 검색바의 "📲 설치" 버튼 또는 브라우저 메뉴 → "앱 설치" → 홈 화면 추가. iOS Safari: 공유 시트 → "홈 화면에 추가". 설치 후 standalone 모드(주소창 없는 전체 화면)로 실행됩니다.

오프라인에서도 마지막 로드된 데이터가 표시되며, 새 버전이 배포되면 토스트로 알려줍니다.

데이터 · 보안

  • 모든 매물은 mock 데이터로, 실제 거래 대상이 아닙니다.
  • 실 법원경매·온비드·국토부 API 호출, 실제 입찰·금전거래·외부 전송은 하지 않습니다.
  • 관심·메모·비교·검색어·본 이력·테마는 자기 폰 브라우저(localStorage)에만 저장되며 서버로 전송되지 않습니다.
  • 외부 라이브러리 0, 외부 API 호출 0 (mock JSON과 RSS만 같은 출처에서 fetch).
  • ⚙ 설정에서 6종 키를 한 JSON 파일로 백업/복원할 수 있습니다 (덮어쓰기·병합 모드).

자주 묻는 질문

Q. 데이터는 실제인가요?

A. 모두 mock 데이터입니다. 실제 매물·시세·실거래·등기부등본과 무관하며, 데모와 UX 테스트 목적으로만 제공됩니다.

Q. 데이터는 어떻게 갱신되나요?

A. GitHub Actions의 daily cron이 매일 새벽(한국 시간 7시)에 mock 파이프라인을 다시 돌려 docs/data/mock_dashboard.json과 docs/feed.xml을 자동 커밋합니다. push 후 GitHub Pages가 자동 재배포합니다.

Q. 폰을 바꾸면 ★/📝가 사라지나요?

A. localStorage는 폰·브라우저별로 분리됩니다. ⚙ 설정 → 백업 파일 다운로드로 JSON을 받아두고 새 폰에서 같은 메뉴로 복원하면 옮길 수 있습니다.

Q. 비밀번호나 로그인이 필요한가요?

A. 필요 없습니다. 로그인·계정·서버 저장이 일체 없습니다.

Q. 광고나 추적이 있나요?

A. 광고·외부 추적 스크립트가 없습니다. GitHub Pages 호스팅 자체의 표준 액세스 로그만 발생할 수 있습니다.

Q. 실제 입찰을 해도 되나요?

A. 안 됩니다. 표시된 모든 매물은 가공된 mock이며, 실제 입찰 전에는 등기부등본·전입세대열람·현장조사·전문가 자문이 필요합니다.

면책

  • 본 대시보드는 데모용 정적 페이지입니다. 표시된 모든 매물은 mock 데이터이며 실제 거래 대상이 아닙니다.
  • 권리분석 표시는 참고용 위험 체크리스트와 추가 확인사항으로만 제공되며, 법률·투자 판단을 대체하지 않습니다.
  • 실제 입찰 전에는 등기부등본·전입세대열람·현장조사 등 직접 확인이 필요합니다.
  • 실 API 연동 시점에는 데이터 제공처 정책과 개인정보 보호 의무를 따릅니다.
